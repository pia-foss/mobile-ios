# PIALocalizations

Swift Package containing all localizable strings for the PIA VPN iOS and tvOS apps.

## Overview

This package is the single source of truth for all user-facing strings, per [ADR-0002](../../ADRs/0002-swiftui-feature-modules-as-swift-packages.md). Feature packages and the main app targets do not bundle their own `.strings` files.

The `L10n` enum is generated by SwiftGen from `Localizable.strings` and provides type-safe access to all strings at compile time.

## Structure

```
PIALocalizations/
├── Package.swift
├── swiftgen.yml                          ← (root repo swiftgen.yml drives generation)
└── Sources/PIALocalizations/
    ├── Generated/
    │   └── L10n.swift                    ← Auto-generated, do not edit
    └── Resources/
        ├── en.lproj/
        │   └── Localizable.strings       ← ~610 keys
        ├── ar.lproj/
        │   └── Localizable.strings
        └── ...                           ← one lproj per language
```

## Usage

Import the module in any target that needs localized strings:

```swift
import PIALocalizations

label.text = L10n.Global.ok
button.setTitle(L10n.Signup.InProgress.title, for: .normal)
```

## Adding or Updating Strings

1. Add the new key and English value to `Sources/PIALocalizations/Resources/en.lproj/Localizable.strings`
2. Add the translated value to all other `{lang}.lproj/Localizable.strings` files
3. Regenerate `L10n.swift` from the repo root:

```bash
swiftgen config run
```

> `L10n.swift` is checked in. Always regenerate and commit it alongside string changes.

## Regenerating L10n.swift

SwiftGen is configured in the root `swiftgen.yml`. Run from the repo root:

```bash
swiftgen config run
```

Requires SwiftGen to be installed:

```bash
brew install swiftgen
```
